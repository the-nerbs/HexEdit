Basic Find Page

 

 

This page allows you to do simple text and hex searches.

 

Find

Type what you want to search for here.  If you type in pairs of hex digits (with optional space separation) then it is assumed you want to do a hex search, and the pairs of hex digits are taken as the hex bytes to search for.  Otherwise it is assumed that you want to search for the characters as text exactly as they were typed.  You can also select something you looked for before by selecting an entry in the drop-down search history list (the same history as is available for the Find Tool).

 

Note that for a hex search the "whole word" and "match case" options are irrelevant and are disabled.  For more advanced hex search options use the Hex Find page.

 

Also note that for a text search the character set used is that in use (ASCII or EBCDIC) in the currently active file.  If you want to explicitly choose a character set or use more advanced options then use the Text Find page.

 

The search algorithm (Boyer-Moore) used is faster the longer the sequence of (unique) bytes entered. So entering a search string of “ABCDEFGHIJKLMN” is better than “ABC” even if “ABC” is sufficient to find what you are looking for. But entering “ABCXDEFXGHIXJKLX” would be no faster than “ABCX”, because of the repeated “X”.

Find Next

Searches for (and selects if found) the next occurrence of the text or hex digits specified in the Find box.

Help

Invokes help for this page.

Whole word

For a text search only a whole word is matched.  The text is not found if it is part of a larger word.

Match case

For a text search the case of alphabetic characters is considered.  If this option is off then the search is case-insensitive.

Direction Up/Down

These radio buttons say whether the search moves forward or backward in the file.

Scope: To Mark

Sometimes it is useful to restrict a search to just part of a file, particularly if the file is very large.  This option means that the search is only done in those bytes from the current cursor position to the mark in the active file (see Set Mark Command).  You can see how many bytes will be searched by viewing the "Distance to Mark" panes of the status bar.

 

This option is disabled if the current cursor position is on the wrong side of the mark (depending on the search direction - see above)

Scope: Whole file

Use this option if you want to search the whole file irrespective of the current cursor position in the file.  Depending on the direction of the search (see above) the file is searched starting at the first or last byte.  After the first successful search with this scope is complete the scope then changes to "To Start of File" or "To End of FIle" (depending on the current search direction) so that you can just hit the "Find Next" button to find the next occurrence.

Scope: To Start/End of File

The scope of the search is from the current cursor position to the end of file (or start of file if the current search direction is upwards).  When the end (or start) of file is reached you are asked if you want to continue the search from the other end of  the file back to the start.

Scope: All Open Files

When this scope is used the current file is searched from the current cursor position to the end (or start if direction is upwards), and if nothing is found then other open files are searched in turn.  Finally, if no occurrences are found in all other files then the start (or end) of the current file is searched back to the initial position.
