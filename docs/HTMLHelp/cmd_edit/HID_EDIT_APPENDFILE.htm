Append Selection (Edit menu)

 

This is similar to the Write Selection to File command except that it appends to an existing file rather than creating a new file, or overwriting a file. This command brings up the standard File Open dialog which allows you to select an existing file to append to. This command is unavailable if there is nothing currently selected.

 

This command was primarily added due to the 16 Mbyte limit of the Windows clipboard. For example, rather than cutting and pasting to build a new file, this command can be used to assemble a new file by adding all the pieces to a disk file. You can then open the new file in HexEdit to check it and make minor changes.  Once you have written to or appended to a file you can quickly append more bytes to the same file using the Append Same File command.

 

Even for chunks smaller than 16 Mbytes it is more memory efficient to create a new file using this command rather than copying and pasting using the clipboard. Copying and pasting many large blocks may use excessive amounts of virtual memory causing your computer to slow down and even crash if virtual memory becomes exhausted.
