Error: Procedure entry point ... not found

 

If you get a message like "The procedure entry point __lc_collate couldn't be located in MSVCRT.DLL" when you run HexEdit it means that the DLL file that is installed on your machine is an earlier version than HexEdit requires.  Under Windows 95/98/ME you may see a message like "A device attached to the system is not functioning ... The BCGCB59.DLL file is linked to missing export MFC42.DLL:6880".

 

HexEdit requires three Microsoft DLLs with the following (or later) dates:

 

MSVCP60.DLL   17 Jun 1998

MSVCRT.DLL   23 Apr 1999

MFC42.DLL   27 Apr 1999

 

These files are usually installed in the Windows system directory.  This is typically (ie by default) C:\WINNT\SYSTEM32 under NT/2000, C:\WINDOWS\SYSTEM32 under XP, or C:\WINDOWS\SYSTEM under Windows 95/98/ME.

 

HexEdit installs MSVCP60.DLL (as long as you left the "Shared DLLs" option ticked in the HexEdit setup) so this is not normally a problem.  If you do have a problem after installing HexEdit (under NT/2000/XP) and you do not have rights to modify the Windows system directory you may need to reinstall while logged in as administrator or have your system administrator install the file.

 

MSVCRT.DLL and MFC42.DLL are not distributed with HexEdit since almost everyone has them.  If you have old versions of one or both of them you can obtain them here.  Again, if you do not have administrator rights (under NT/2000/XP) you may need to get your system administrator to install them.
