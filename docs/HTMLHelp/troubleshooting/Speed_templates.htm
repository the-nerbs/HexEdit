Speeding up templates

 

Loading a large file that uses a template can be very slow, since HexEdit must process the whole file before it can display the tree view. (Of course, the speed is somewhat dependent on the template in use and the actual data of the file.)  Here are a few ways to speed up the process.

 

First if the template uses arrays ("for" elements) where the elements are of fixed size then HexEdit does not need to process every element of the array.  Instead it can display the first few elements of the array and then a icon representing the rest of the elements.  You can reduce the number of fixed-size array elements that are processed with the "Number of fixed array elements" field in the Display Options page.  (For some templates this may cause errors as elements that are not processed cannot be used in expressions.)

 

The only other way to speed up templates is to modify the template (XML) file itself.  For example, you can remove domain attributes from data elements, especially if the data element is in a large array.  The repeated evaluation of the domain expression will slow down the template processing.

 

If a template has become complicated and slow due to handling the differences between different versions of a file format you can split the template up into several to handle each of the file versions.  This will also make the templates easier to use and maintain.  Of course, in each template you should do a domain check on the version field to make sure the right template is used with the right file version.
