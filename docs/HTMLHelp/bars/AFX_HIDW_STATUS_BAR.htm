Status Bar

 

 

The status bar is displayed at the bottom of the main HexEdit window. To display or hide the status bar, use the Status Bar command in the View menu.

 

The left area of the status bar describes actions of toolbar buttons or menu items as you move the mouse over them or use the arrow keys to navigate through menus. It also displays the occasional status or progress message depending on the current actions being performed.

 

The right area of the status bar has several panes or indicators which have the following uses:

 

Indicator

Description

Compare count

If there is a compare view open for the current file then this shows the number of differences found (or a progress bar if the comparison is still in progress).

Search count

Shows the number of occurrences of the current search text (in the file open in the active window).  If background searching is off (see the System Page of the Options dialog) it says “OFF”.  If there is a background search in progress then it shows the progress of the search (progress bar and a percentage complete).  If there are more than 9,999,999 occurrences of the current search string then “TOO BIG” is displayed instead.

Byte values

The 2nd indicator displays the values of the byte at the current cursor position in the active window.  This shows the value of the byte in hex, decimal, octal and binary.  It also shows the ASCII or EBCDIC character that the byte represents, depending on EBCDIC mode.  To see more information about the byte(s) after the cursor and the file in the active window double-click on this indicator to open the Properties Dialog.

Hex Offset

Indicates the numbers of bytes from the mark to the cursor in hex.  See the Mark Position command.
This pane is shown in red (or whatever color is used for hex addresses in the current color scheme).

Decimal offset

Indicates the numbers of bytes from the mark to the cursor in decimal.

his pane is shown in blue (or whatever color is used for decimal addresses in the current color scheme).

File Length

Shows the file length.  The length is shown in decimal or hex depending on how addresses are being displayed in the active window (see Decimal Addresses command ).  If bytes have been inserted/deleted it also indicates the difference in length from the on-disk (last saved) version.  When editing disks (raw disks or volumes) this shows the current sector and the total number of sectors.

LE/BE

Indicates whether changes to the files are made in little-endian or big-endian mode.  To toggle just double-click or use the Big-endian checkbox in the Calculator.  The big-/little-endian mode has no effect on the display; it only affects how file access from the Calculator is performed and how some commands in the Operations menu work.

RO/RW

Indicates read-only or read-write mode.  To toggle between read-only and read-write just double-click this indicator or use the Allow Changes command.

OVR/INS

Indicates overtype or insert mode.  To toggle between overtype and insert modes double-click this indicator or use the Insert Mode Toggle command.  Many binary file formats cannot tolerate insertions/deletions so you should not use INS mode.

REC

Indicates when macro recording is active. To start or stop recording double-click this indicator or use the Record command.

CAP

Indicates the state of the Caps Lock toggle.

NUM

Indicates the state of the Num Lock toggle.
