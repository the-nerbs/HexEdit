Export Motorola S3 Records command

 

This command writes the current selection to a Motorola S format file.  It only writes S3 data records which have a 32 bit address, so this command is disabled if the address of the last byte is greater than 4,294,967,295 (which may depend on how the start address is calculated -- see below).  It is also disabled if there is no selection and no highlighted bytes.

 

This command brings up the Export Motorola S Dialog which allows you to save the file using any valid file name. If the file already exists you are prompted to overwrite it.  A Motorola S format file is a text file.

 

New in HexEdit 3.2 is the ability to export discontiguous records.  To enable this select the "Export highlighted areas only" if the Save As dialog.  This checkbox is disabled if no part of the file is highlighted.

 

Each S3 record contains an address and a certain number of bytes stored as text along with a checksum.  Unless the aforementioned option is selected, HexEdit will only create S3 records that are contiguous, and will always generate correct checksums.  The number of bytes per record and how the starting address is generated are determined by the Export section of the System Options page of the Options dialog.

 

An S0 (start) record is written at the start of file and an S5 (record count) record is written at the end of file.
