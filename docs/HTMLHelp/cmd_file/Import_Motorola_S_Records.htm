Import Motorola S Records command

 

This command reads an Motorola S file and inserts it at the current cursor location.  The command is unavailable if the file is read-only.  The Import Motorola S Records dialog is displayed to allow you to select a file to insert.

 

This command handles all data records (S1, S2 and S3 records) as well as the record count (S5) record.  All other records are checked and thrown away.  An error is generated if the checksum of any record is in error, the S5 record count is incorrect.  Although the S5 record is checked if it is missing a warning is displayed but the file is still processed.

 

The behaviour depends on the use of the "Overwrite at actual import record address" option as described here:

 

If the option is off then the file must be in insert mode and the imported bytes are inserted into the file at the current cursor position.  An error occurs if the addresses of data records are not contiguous.  The inserted bytes are selected so you can easily see which bytes were added.

 

If the option is on the imported bytes are placed at the address of the import record.  The addresses do not have to have any order.  The imported bytes are not selected since the records may not be contiguous.  

If an address is past the end of the file then it is ignored.  The data replaces the data of the file, hence the file length never changes as a result of the operation.  (The existing bytes are overwritten but can, of course, be recovered by undoing the operation.)
