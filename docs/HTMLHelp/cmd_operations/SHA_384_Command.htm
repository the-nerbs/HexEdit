SHA 384 command

 

This performs a cryptographic checksum (also called a message digest or one-way hash function) on the currently selected bytes using SHA 2.  The result is a 384-bit integer which is placed in the Calculator and (optionally) displayed as hex (as is conventional) in a popup window.

 

If not already in that mode the calculator is switched to hex and to use (at least) 384 bits.

 

To copy the result from the calculator just press Ctrl+C.

 

It is strongly believed to be computationally infeasible for someone to calculate a set of bytes that would generate a specified SHA2 value.  In other words if you know the SHA2 value for a set of bytes you can be sure they have not been tampered with by checking that they still generate the same value.  (This contrasts with CRCs which, although excellent for detecting accidental changes to data, are no guarantee that the data has not been deliberately tampered with.)

 

Like MD5 and SHA1, SHA 2 can be used to verify that files downloaded from the Internet have not been modified.  SHA 2 (and SHA 3) are considered more secure than SHA 1 and especially MD5.

 

To generate the value for the whole file using HexEdit just select the whole file (eg Ctrl+A) before using this command.
