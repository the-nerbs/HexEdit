Encrypt command (Operations menu)

 

Encrypts the selected text using the current encryption algorithm (see Encryption Algorithm) and the current password (see Encryption Password).  If no password has been entered, or the Clear Password command has been used then the user is prompted to enter a password in the Encryption Password Dialog - a feature that is very useful in macros.  Once encrypted the only way to get back the data is to decrypt using the same algorithm and password (see Decrypt).

 

If there is no selection, or if the current encryption algorithm is the "HexEdit internal algorithm" and the selection is not a multiple of 8 bytes, then you cannot encrypt the selection.  If the current algorithm is a CSP block cipher then the length of the selection will be increased by a few bytes after encryption.  Hence if you are in overtype mode HexEdit will prompt you to change to insert mode.

 

In HexEdit 3.2 you can now encrypt or decrypt a selection larger than 4 GBytes (unless using the internal algorithm).
