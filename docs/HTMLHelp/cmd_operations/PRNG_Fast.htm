Generate Fast Random Numbers (Operations menu)

 

This command generates random integers using the algorithm provided by the C standard library "rand" function included with the Microsoft Visual C++ 6 compiler.  This is a linear congruential algorithm that produces quite good random numbers quickly.

 

The drawbacks of this algorithm are that the least significant few bits can be decidedly non-random and the cycle is fairly short (about 2^31).  The first problem is not important in HexEdit as each random number is use to generate one byte and the least significant bits are not used.

 

The second problem means that if you generate more than about 32Kbytes of random numbers the sequence will repeat.  Depending on the use of the numbers this may or may not be a problem.  The numbers will still pass statistical tests of randomness but the predictability means they are not appropriate for encryption/security type applications.

 

For better (but slower) numbers with a much longer cycle use the Generate Good Random Numbers command.

 

To generate a new sequence of random numbers use the Random Seed command before repeatedly calling this command.  To generate a reproducible sequence of random numbers load a specific value into the calculator and call the Seed Random Number Generator from Calculator command.
