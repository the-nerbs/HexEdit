Step 1. Getting Started

 

         

 

First we will create a simple macro just to get started.  (You can skip this step if you are already familiar with using keystroke macros.)  Then we will quickly move on to discuss what the main macro of the tutorial does and how it will be implemented.

 

A Simple Macro

 

When macro recording is turned on, you can use HexEdit as normal, but all your keystrokes, as well as menu commands, many mouse actions etc are recorded just like using a tape recorder.  To turn macro recording on and off you use the Tools/Record Macro menu item.  While you are recording the "REC" indicator is turned on in the status bar.

 

Other ways to start/stop recording include the Record/Stop button on the Standard toolbar or the F7 key (assuming you have not customised the toolbar or the keyboard).  You can also double-click the "REC" indicator in the status bar.

 

1. Open any file and press Ctrl+Home to move the cursor to the start of file.

2. Select Tools/Record Macro (or press F7) to turn on macro recording.

 

 

Figure 1.1 Macro recording is indicated in the REC box of the status bar.

 

3. Press the End key to move to the end of the first line.

4. Press F7 again to turn off macro recording.

 

You have now recorded your first macro that moves to the end of the current line.  Try using it now:

 

5. Press the right arrow to move to the start of the 2nd line.

6. Press F8 to run the macro.  The cursor should move to the end of the 2nd line.

 

Remember that the macro plays back your commands exactly as you recorded them.  This means that a macro may not behave exactly the same when played in different circumstances or with different data.  For example, the macro you created above will move a different number of bytes depending on the length of the line (and hence also the window width if AutoFit mode is on).

 

Creating a complicated keystroke macro requires careful thought about all possible situations it could be used in, and usually involves some trial and error.

 

Now it's time to create a much more powerful, and useful, macro.

 

What the macro will do

 

HexEdit macros are a very powerful tool that can perform many tasks that would normally require programming.  The macro in this tutorial is designed to perform a task that was actually an enhancement request from a user of HexEdit.  (We get many requests for new features that can often already be performed using keystroke macros.)

 

Basically the macro is an implementation of "one-time pad encryption".  That is, given a file it generates another file of the same length which is filled with random bytes.  It then XOR's the original file with each corresponding byte from the random file.  This gives a very effective form of encryption as long as the values of the random file are never revealed.

 

Of course, the random file (or information on how it was generated) must be retained if you ever want to decrypt the data again.  For important data it should be kept in at least two physically separate, secure locations.  You can then freely copy the encrypted file with the knowledge that nobody can decipher it unless they have access to your random file (or know exactly how the random numbers were generated).

 

If other people have (or may have) access to your computer or your drive (eg, across a network), then you will want to copy the random file to removeable medium like a diskette, and delete it from your hard disk.  Note that HexEdit overwrites the original file in place so there should be no need to delete it unless you have copied it elsewhere on your hard disk, or you have configured HexEdit to create a backup copy.

 

When you delete the random file or backup copies of the original file, they may still be able to be recovered from the "free space" of the disk.  To completely remove the data from your disk you need to use a facility like Norton Utilities "WIPEFILE" command.

 

How the macro works

 

The macro works by using the "New File" dialog to create a file of the same size as the original file but filled with random bytes.  It does this by getting the size of the original file into the calculator then using the "Take size from calculator" option for the size of the new file with the random fill option.

 

Then the macro calls a "nested" macro to repeatedly XOR the bytes of the random file with the original.  To do this it (the nested macro) reads a byte from the original file into the calculator, XOR's it with a byte from the random file, then writes the new value back to the original file.

 

The nested macro must also move the cursor (in both files) to the next byte so that, as it is called repeatedly, it processes the whole file.  It does this by loading the cursor address into the calculator (using the "Cursor" button), incrementing it and then updating the cursor address (using the "Set" cursor button).

 

We have to create the nested macro first as it needs to be present on disk when we create the main macro.  In the next step of the tutorial we will do just that.
