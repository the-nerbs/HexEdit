Step 5. Saving and Testing the Macro

 

         

 

All that remains is to save the macro and do some tests to make sure it is working OK.

 

Save the macro

 

1. Open the Macro page of the Options dialog and click the "Save Current Macro" button.

2. Enter a name for the macro of "OneTimePad".

3. Change the default number of replays to one.  Leave the halt condition to be minor errors.

4. In the comment field, enter a description of what the macro does and any other relevant info.

 

 

Figure 5.1 The Save Macro dialog.

 

5. Click the "Save" button to save the macro and click the OK button to close the Options dialog.

 

The macro is now finished and saved!

 

Testing

 

Finally, we will give the whole macro a test to make sure it works.

 

6. Close any open file(s) and create a large file filled with null bytes (00) and make sure it is RW (not readonly).

7. Use Tools/Play... to open the Replay Macro dialog.

8. Select the "OneTimePad" macro from the list and click OK.

9. The macro displays a message.  Click OK to continue.

 

We filled the file to be encrypted with null bytes so that after the encryption it should be the same as the random file.  (XOR'ing with zero gives the same value.)  We can check that the files are the same using the HexEdit file comparison facility.

 

10. Press Ctrl+Home in both files to go to the start.

11. Use Tools/Compare (or Alt+C) to compare the files.

 

 

Figure 5.2 The files are identical.

 

Note that this is not a definitive test, but gives a good indication that the macro is working properly.

 

Finally, we leave it to the reader to create a macro to decrypt the file.  A clue is that to get back an XOR'ed value you just need to XOR with the same value again (see Boolean Operations).
