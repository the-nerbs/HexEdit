Aerial View Page

 

 

This page allows you to change global options relating to aerial view windows.

 

Maximum aerial view bitmap size

This is the largest bitmap that HexEdit will create in memory for the aerial view display for any single file.

 

If the file is too big then the aerial view bitmap will be subject to pixel merging. That is each pixel will represent 2, 4, 8, 16, ... bytes of the file to keep within the limit. Note that even in this case some information is retained as the pixel for any group of bytes (2, 4, 8, etc) has a pixel color which is a combination of the colors for all the bytes of the group (as determined by the color scheme in effect).

 

The exact size of the when pixel merging occurs is one-third of the maximum aerial view bitmap size. (Each pixel in the bitmap is 24-bits or 3 bytes.) For example, if the maximum aerial view bitmap size is 1.5 GBytes then when a file greater than 0.5 GBytes will require pixel merging in the aerial view display.

 

You can see how many bytes are merged into one pixel by double-clicking any pixel in the aerial view. This will select the corresponding byte(s) in the hex view. If pixel merging was required the number of bytes thus selected will be a power of 2 (2, 4, 8, 16, ...).
