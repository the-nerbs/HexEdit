Color Options Page

 

 

This page allows you to change the colors used to display the active file.  You can also create and use new color "schemes", or delete and modify existing schemes.  A very useful feature is the ability to display different groups of hex values in different colors, which can be useful for quickly spotting patterns and irregularities.

 

 If you modify a color scheme all files which use that color scheme will be affected (not just the active file).

 

Schemes

 

The left side of the page displays the current schemes with buttons to create new schemes, and delete or reset the currently selected scheme.  The top four schemes are standard schemes that cannot be deleted.  To change the colors of a particular scheme (see below) first select it in the list.

 

The top four schemes are "standard" schemes used specially to display different characters sets.  So if the current character set is ASCII (see Char Set ASCII) and the current scheme is "ASCII default" then changing to the EBCDIC character set (see Char Set EBCDIC) automatically changes to the "EBCDIC default" color scheme.

 

You can modify the colors for the standard schemes.  To restore the factory default colors use the "Reset" button.

 

Current Scheme

 

The right side of the page displays the available colors for the currently selected scheme.  The top 10 colors are standard ones that cannot be deleted, or reordered.  Below the standard colors are one or more entries that specify color groups that can be created, deleted and reordered.

 

To use different colors you need to split the range of byte values (0-255) into groups. A group can consist of individual values or ranges. For example, to use the values found in normal ASCII text files use “9,10,13,32-126”, which are the values of tab, line-feed, carriage-return and the range of normal display characters. As well as the values each group has an associated name and, of course, a color.

 

When displaying HexEdit scans the groups from the top and uses the color of the first group the value is found in. If a value appears in more than one group the color from the first group is used. You can use the arrow buttons to change the order of the groups. Also if a value is not assigned to any group then that value is not displayed. (Actually it’s just displayed using the background color.)

 

 

The new scheme button invokes the New Scheme Dialog which allows you to create a new scheme based on any existing scheme.

 

To delete a scheme select it in the list below and click this button.  You cannot delete the first 4 standard schemes but you can, of course, change their colors.

 

Reset

This button changes all the colors of the currently selected scheme back to the default.  The default colors for the first four (standard) schemes are all slightly different.  Reseting any other scheme sets it to be the same as the default "ASCII default" scheme.

 

List of Schemes

Select a scheme from the list before you delete, reset or modify it.

 

Selecting a scheme and clicking OK selects that scheme for use in the active window, if any.

 

Name

The description of the currently selected group. You can also type the name of a new group to be added here.

 

List of Names

The top 10 entries are for standard colors that every color scheme must have.  Below that are the names of all the current groups. To change the color of any of the entries first select it in this list before using the color selection tool (below).  Note that since HexEdit 3.0 there are separate colors for the mark and bookmarks.

 

Background

The background color of the window, usually white or a very light shade.

Mark

Background color for the byte at the "mark".  See Set Mark Command.

Highlighting

Background color of bytes that have been highlighted.  See Highlighter.

Bookmarks

Background color of bytes that have been bookmarked.  See Edit Bookmarks.

Search occurrences

Background color of all occurrences of the current search text (when background searches are enabled).  (See Background search option in System Options page.)  Also used for search occurrence progress bar in the status bar.

Change tracking

Used to show replacements, insertions and deletions.  See Track Changes command.

Compare Tracking

Used to shows differences with the current comparison file (replacements, insertions and deletions).

Address/ruler background

Background color of the ruler (top of window) and address area (left side).

Sector boundary

Used to show the boundary of disk sectors and the background color for bad sectors if the "Sector Bounds" option is on in the Window Display Options page.

Hex addresses

Addresses are shown in this color if the Decimal addresses toggle is off.  Also used for hex addresses in the status bar and hex jump tool and many other places.

Decimal addresses

Addresses are shown in this color if the Decimal addresses toggle is on.  Also used for decimal addresses in the status bar and decimal jump tool and many other places..

Standard Colors and Uses
 

Below the Decimal Addresses color are one or more colors used to display the bytes of the file (both hex and characters).  Select one of these before adjusting its color or range of values using the controls below.

Color

Select a color from the drop down list to change the color of the currently selected entry in the list above.  You can select one of the 20 colors or "Automatic" to choose a color consistent with your "Windows Display Appearance Scheme".  Select "Other" to see a dialog with two pages: the "Standard" page allows you to easily select from 140 standard colors; while the "Custom" page allows you to enter any color based on RGB or HSV values.

 

Values

Enter the values for the current or new group. Values are in decimal and comma-separated; a range of values may be entered by typing two values separated by a dash or a colon.

 

The new color group button adds a group to the list with the name currently in the Name field (see above), and the currently selected color and range of values. You cannot Add a group with the same name as an existing group.

 

This removes the current group from the list. Before removing you must select the group to remove from the list.  You cannot remove any of the first 10 standard items from the list.

 

If you accidentally remove a group, simply click the new color group button before you do anything else. This reinserts the same group. You may then need to use the up arrow button to reposition the group where it was before.

 

The order of the groups is significant. The up arrow allows you to move the current group up in the list.  You cannot move any of the first 10 standard colors.

 

The down arrow allows you to move the current group down in the list.
