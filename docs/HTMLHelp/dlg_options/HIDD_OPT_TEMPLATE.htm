Template Options Page

 

 

This page allows you to change some options for templates that apply to all files.

 

The default template display options are no longer settable here.  Use the Save As Default button on the Document General Options page after setting the template display from the Template View Display.

 

Number of fixed array elements

For large arrays (with constant element size) it may be faster to display only the first few elements of arrays.  This setting allows you to change how many elements are displayed.

 

Change Templates Folder

Click this button to change the location that HexEdit uses to load and save templates  By default this is under your Application data area.  For example in Windows XP this is something like:

 

C:\Documents and Settings\<login>\Application Data\ECSoftware\HexEdit

 

Under Vista it is something like:

 

C:\Users\<login>\AppData\Roaming\ECSoftware\HexEdit

 

Note that if you move the location of the folder you need to copy several required template files into it (as well as any other templates that you want to use).  You can copy them from the previous templates folder, but they are also present in the folder where HexEdit is installed (usually C:\Program Files\HexEdit).

 

The files you need are BinaryFileFormat.DTD and Default.XML.  If you use the C/C++ source parser you also need _standard_types.XML, _common_types.XML, _windows_types.XML, and _windows_constants.TXT (all these file names start with an underscore).  If you have saved constants and/or types from earlier parses you can copy these in the files _custom_types.XML and _custom_constants.TXT.

 

Default format strings

These strings determine how data elements of a template are displayed if the template file does not explicitly specify the format.  For example, you may want to see unsigned integers displayed in hex (%x) rather than decimal (%u).  Refer to C language printf format specifications for more information, and strftime for date formats.
