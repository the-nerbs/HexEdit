HexEdit Explorer Page

 

 

This page allows you to change options used in the HexEdit Explorer.

 

Wipe Strategy

You can delete and wipe a file by right-clicking it in the Explorer window and selecting the option from the context menu (assuming the Use Custom Context menu option is selected - see below).

 

When you wipe a file all bytes are overwritten to remove any data they contain. However, just overwriting a disk block with different data does not (in theory) obliterate all trace of the previous data. It's possible that a criminal with special hardware could recover some of the data even though it has been overwritten. Consequently "wipe" tools usually provide different strategies which trade off speed and security. For almost everybody the "Good" strategy should be more than sufficient as it overwrites each byte three times (all bits on, then a random value, then all bits off). The "Thorough" strategy is slow and is based on a security standard (TBD reference needed).

 

WARNING: HexEdit makes every effort to force the data to be written to the physical disk between passes. However, some hardware (controllers or drives) may cache data. It's possible, though unlikely, that with poorly designed hardware even using the "Thorough" strategy may only write to the physical disk once.

 

As well as wiping the current data of the file HexEdit also extends the file before wiping so that any data in the last cluster of the file after the current end of file is also wiped. This ensures that old data is wiped if the file has been shortened. But note that if a file is shortened such that blocks from the end of the file were added back to the partition's free list then these blocks will not be erased and data from these blocks may be recoverable. If this case is important to you then you should obtain a tool that can "wipe" all free blocks on the disk.

 

Not Indexed Attribute

Every file on disk has a set of flags or attributes. These are displayed in the Attributes column in the Explorer tool window. Attributes that have always been supported in Windows (and MSDOS) are R (read-only), H (hidden), S (system) and A (archived).

 

Windows now supports other attributes like like compressed and encrypted. Another attribute is to mark a file to not be indexed. (Windows has an indexing service that scans and "indexes" files in the background for faster searches.) Some people find it confusing to have a flag that indicates that something does not occur so HexEdit provides two options:

 

I is displayed in the file attributes to indicate that a file will be indexed - ie, the Not Indexed flag is off

 

N is displayed in the file attributes to indicate that the file will not be indexed

 

Keep tree synchronized with active folder

This option updates the Explorer folder view (the tree view to the left of or above the file view) whenever the currently displayed directory of folder is changed. This happens whether you double-click a folder name, type in a new folder name

 

Use Custom Context menu

When this option is selected HexEdit will display a custom context menu when you right-click any file.
