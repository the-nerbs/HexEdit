Integer Values Page

 

 

This page shows, and allows you to edit, the decimal values of the byte, word, double word and quad word of the data following the current cursor position in the active window. Some or all of these fields may be disabled if there are not enough bytes before the end of file.

 

These values are useful if the file you are using is used to store binary numbers, and you need to see or change the actual (decimal) values of those numbers. These values can be displayed and edited as signed or unsigned and as big-endian or little-endian depending on the check box options chosen. (Typically for signed values you would use 2’s complement format, the one’s complement and sign+magnitude formats are not common nowadays.) The big-endian option is useful for viewing files from certain “foreign” sources such as many UNIX systems.

 

You can change any of the values by simply typing a new value and pressing Enter. If you make a mistake or change your mind just press Escape before pressing Enter. (If you pressed Enter you can still undo your changes as normal.) What is stored to file depends, of course, on the big-endian check box and the currently active Format button. If you enter a value that is too big then excess high bits are discarded. Note that you cannot change any values if the window is read-only or there are not enough bytes before the end of file.

 

8 Bit

The decimal value of the byte. This field uses values in the range 0 to 255 (unsigned), –128 to 127 (2’s complement), or +/-127 (1’s complement/sign+magnitude).

16 bit

The decimal value of the two bytes following the cursor. This field uses values in the range 0 to 65535 (unsigned), –32768 to 32767 (2’s complement), or +/-32767 (1’s complement/sign+magnitude).

32 bit

The decimal value of the four bytes following the cursor. This field uses values in the range 0 to 2^32-1 (unsigned), –2^31 to 2^31-1 (2’s complement), or +/-2^31-1 (1’s complement/sign+magnitude).

64 bit

The decimal value of the eight bytes following the cursor. This field uses values in the range 0 to 2^64-1 (unsigned), –2^63 to 2^63-1 (2’s complement), or +/-2^63-1 (1’s complement/sign+magnitude).

Big endian

Select this option to view the numbers with big-endian byte order. When selected the most significant byte is considered to be the left-most byte (ie, the byte with the lowest address). This option is off by default since data created on MSDOS/Windows systems is typically little-endian.

Unsigned

Select this option to view the numbers as being unsigned.

2’s complement

Select this option to view the numbers as being signed in 2’s complement notation. Negative numbers have the high bit on – to obtain the inverse value flip all the bits and add one. The high bit is the most significant bit of the most significant byte (see the Big endian option above).

 

You will typically use this option for signed numbers as the other signed formats are not used in an Intel/Windows environment.

1’s complement

Select this option to view the numbers as being signed in 1’s complement notation. Negative numbers have the high bit on. Invert all the bits to find the corresponding positive number.

Sign and magnitude

Select this option to view the numbers as being signed in sign and magnitude notation. Negative numbers are the same as positive numbers but with the high bit on.
