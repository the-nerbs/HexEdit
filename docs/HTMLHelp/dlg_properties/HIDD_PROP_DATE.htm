Date/Time Values Page

 

 

This page shows, and allows you to edit, several date formats using the data following the current cursor position in the active window. These fields may be disabled if there is no active window or there are not enough bytes before the end of file.  (The number of bytes used by the currently selected format is displayed under the word "Format".)

 

You can change the values to any valid date for the selected format using the date/time controls or by clicking one of the four buttons. However, you can't make changes if the data is read-only.  If you accidentally change a date/time value you can use the "undo" facility to restore the previous value as usual.

 

All formats support both big- and little-endian, except for MSDOS filesystem date/time format which is a strict bit-pattern.  The big-endian option is useful for dates from sources such as Macintosh, Amiga, IBM mainframe and many UNIX systems.

 

Note that there may be variations in the way dates are interpreted.  So the value you see may not be exactly what you expect.  For example, times are usually stored as UTC, but sometimes are stored as local time due to software bugs or system administration problems.  Some formats can be used to store date-only or time-only values.  Also the exact values of 1st, last and invalid date values are vague or unspecified for some formats.

Format

Several dates formats are supported.  The first format is by far the most common format for time_t as supported by most versions of UNIX and even in most C compilers running on other operating systems.  The two time_t formats below it were used by different versions of Microsoft's 16 bit C/C++ compiler, but it is unlikely that you will encounter time_t data in these formats.  The other time_t format is apparently used under some versions of UNIX.

 

HexEdit also now supports a 64 bit version of time_t, which is recently implemented in several compilers (but not yet standardised).  The problem with a "normal" time_t is that it only handles dates up sometime in 2038.  The 64 bit time format uses the same value as time_t (seconds from 1/1/1970) but uses a 64-bit integer.  This means dates far into the future can be stored.  The time64_t date format provide here is as implemented in Visual C++ 7 (VS.net 2002) and later, and is only defined to be valid in the range of times from the 1st second of 1/1/1970 to the last second of 31/12/2999.

 

The formats following the time_t formats are specific to Microsoft operating systems.  The OLE DATE format was created for use by OLE and VB -- it is the number of days since 31/12/1899 expressed as an IEEE 64-bit floating point number.  The SYSTEMTIME structure was invented for NT's use of times.  The FILETIME (essentially a 64 bit integer representing nano secondss since 1/1/1601) was invented for storing file times under NT's filesystem (NTFS).  Similarly the MSDOS format was invented for MSDOS file times.

Big endian

Select this option to view dates with big-endian byte order. When selected the most significant byte is considered to be the left-most byte (ie, the byte with the lowest address).

 

This option is off by default since data created on MSDOS/Windows systems is typically little-endian.  For SYSTEMTIME this reverses the byte order of all the fields of the structure individually.  This option is disabled altogether for MSDOS times since these have a fixed format.

Date and Time (Long Format)

Immediately below the Format drop-list the date and time is displayed in the current long format (as selected in the Control Panel).  This is typically an unambiguous and easy-to-read date and time including the day of the week.

Date

The date control allows you to view the current date and change it.  You can change the date fields by typing them in directly or by selecting a date using the drop-down calendar control.  The tick-box to the left indicates whether the date is valid.

Time

The time control allows you to view the current time and change it.  You can change the components of the time by typing new values or by using the spin control at the right side of the control.  The tick-box to the left indicates whether the time is valid.

1st

Click this button to set the date/time to the first valid time supported by the currently selected format.  For some formats (such as OLE) the exact 1st valid time is ambiguously or vaguely documented, and is taken to be the latest of several different values.

Last

Click this button to set the date/time to the last valid time supported by the currently selected format.

Now

Click this button to set the date/time to the time right now..

Invalid

Click this button to set the date/time to an invalid value.  Note that for some formats this may not be a documented invalid value but simply a value that lies outside the set of valid values..
