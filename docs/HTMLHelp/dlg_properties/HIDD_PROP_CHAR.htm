Byte Properties Page

 

 

This page shows information about the byte at the current cursor position of the active window. It also allows you to change the value as an unsigned decimal, octal or binary value. Note that all values are for the current byte at the cursor except for the Unicode field which uses two bytes starting at the cursor position. The Unicode field is disabled under Windows 95/98 as these systems cannot display Unicode characters.

Hex

The hexadecimal value of the byte from 00 to FF.

Decimal

Displays the unsigned decimal value of the byte from 0 to 255. To change it type a new value and press Enter (or press Escape to abort changes). Numbers above 255 are truncated. Negative values can be entered in the Byte field of the Decimal Values page.

Octal

Displays the octal value of the byte from 000 to 377. To change it type a new value and press Enter (or press Escape to abort changes).

Binary

Displays the binary value from 00000000 to 11111111. This field is mainly useful for examining the individual bits of the byte. “0” means the bit is off, “1” means it’s on. To change it type a new value and press Enter (or press Escape to abort changes).  Double-click any bit to flip it.

ASCII/ANSI

The ASCII character at the current cursor position. If the high bit is set (ie, the decimal value is greater than 127) then this is actually the ANSI character. For values less than 32, this field displays the name of the ASCII control character.

X-Byte Char Set

This is the character from the character set determined by the currently selected code page. The character set can be single-byte or multi-byte.  For example, many Asian languages have double byte character sets (DBCS) which are an extension of ASCII.  If a byte is one of several special lead-in byte values then a 2nd byte is used to determine the exact character.  This allows more than the 256 values that can be stored in one byte to be used while allowing backward compatibility with ASCII.

 

If the code page is for a multibyte character set (MBCS) then the displayed character is taken just from the current cursor position if only it is not a lead-in byte.  Otherwise the character displayed may also depend on the byte(s) following byte at the current cursor position.

Code Page

Shows a list of all available code pages.  The current selection determines how the character to the left is displayed.  Windows may include code pages for many European (single byte) character sets and Asian (often double byte) character sets, which are extensions of ASCII.  There may also be EBCDIC code pages available.

Unicode

This is the Unicode character representing the two bytes following the cursor.

Font

Use this button to select the font used for Unicode and multibyte characters set characters.  The font selected is remembered for the next time you run HexEdit .
