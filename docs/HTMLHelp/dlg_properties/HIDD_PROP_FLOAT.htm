Real Values Page

 

 

This page is used to view or edit the bytes following the current cursor position as a real (ie, floating-point) number. You can view and change the value for several formats in either big-endian or little-endian format. If there are not enough bytes before end of file the fields may be blank (ie 4 bytes for 32-bit floats, 8 bytes for 64-bit floats or 16 bytes for C# Decimals).

 

The exact formats are described in the Floating Point Numbers topic but each can be decomposed into a mantissa and exponent.  Note that some formats define a separate sign bit but for simplicity this is included as the sign of the mantissa.  Also some format have a unsigned (biassed) exponent - in this case we subtract the bias to obtain a signed exponent.

 

IEEE formats also have special values for +/- Infinity and an invalid number (NaN).

Value

This is the full value which you may change (if the file is not read-only).  For very large or small numbers it will be displayed in scientific notation - using a (signed) trailing decimal exponent preceded by the letter “e” - unless the format is C# Decimal.  Note that the value may not be exact (except for C# Decimal) as not all binary floating point number can be exactly represented in decimal.

 

To change the value type a new number and press Enter (or press Escape to abort changes). The new value will be reflected in the file (the changed bytes are selected).  The exact bytes inserted depend, of course, on the format and big-endian status.  You will be told of you enter an invalid number but number very close to zero may just be stored as +/- zero, and very large (positive or negative) numbers may be stored as +/- Infinity for IEEE formats.  Also for IEEE number you can also enter the special values “Inf” or “+Inf”, “-Inf” and “NaN” (for Not a Number).

Mantissa

For IEEE “normalised” numbers the mantissa is a positive or negative value between 0.5 and 1.0. As the values approach zero - ie, the exponent becomes smaller – “denormalised” numbers are allowed which have a smaller mantissa.

 

For IBM FP numbers the mantissa is a positive or negative value between 0.0625 and 1.0.

 

For C# Decimal the mantissa is any valid signed (2s-complement) 96-bit integer - ie anything between -39614081257132168796771975168 and 39614081257132168796771975167.

Exponent

For IEEE numbers this is the (power 2) exponent. For normal numbers it is an integer from 127 to 126 for 32 bit values or –1023 to 1022 for 64 bit values. It is –128 (or –1024) for zero and small denormalised values and 127 (or 1023) for infinity and NaNs.

 

For IBM FP numbers this is the Excess-128 exponent (power 16), which is a small, integer between –64 and 63.

 

For a valid C# Decimal this can be any value from -28 to +28.

Big endian

Select this option to view the numbers with big-endian byte order. When selected the sign bit is in the left-most byte (ie, the byte with the lowest address). This option is off by default since data created on MSDOS/Windows systems is typically little-endian.  But note that IBM floating point values are usually big-endian.

Format

   IEEE 32 bit

Select this option to view the value of the IEEE 32 bit (4 byte) number.

    IEEE 64 bit

Select this option to view the value of the IEEE 64 bit (8 byte) number.

    IBM FP 32 bit

Select this option to view the value of the IBM 32 bit (4 byte) number.

    IBM FP 64 bit

Select this option to view the value of the IBM 64 bit (8 byte) number.

    Real48

Select this option to view the value as a 48 bit (6 byte) real number.  This format was called Real in Turbo Pascal and Real48 in Delphi.

    C# Decimal

Select this option to view the value as a .NET CLR Decimal (16 byte) number.
