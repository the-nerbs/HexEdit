How Do I Search Through Several Files?

 

HexEdit does not provide a facility for searching through multiple files on disk (like the Developer Studio "Find in Files" command).  But you can effectively do the same things by opening all the files you want to search in HexEdit and searching with background searches turned on.  Since opening a file in HexEdit does not read any of the file off disk (except that needed to be displayed in the window) there is little performance penalty in opening the files to search before starting the search.

 

Opening the Files to Search

 

You can easily open a large number of files using multi-select in the File Open dialog.  Just Ctrl+click on each file you want to open, or use Shift+click to select a contiguous range of files.  Don't forget that you can filter the types of files you see to make it easier to select the ones you want by using the filter list (Files of type drop down list).

 

You can also easily open multiple files from the HexEdit command line using wildcards.  If an instance is already running (and "Only Permit one Instance of HexEdit to Run" is selected in the System Options page) then the files are opened in the running instance of HexEdit.  For example, to open all .BMP and .GIF files use:

 

HexEdit  *.BMP  *.GIF

 

If you want to open all files of a certain type including files in sub-directories (or sub-folders as they are now called in Windows) you can use the Windows search facility to find the files.  For example, you can search for all .BMP files by specifying *.BMP in the filename and the directory you want to search in the "Look In" box.  Make sure you have the "Search sub-folders" option on and click the Search button.  When the search is complete select all the found files in the search window and drag them to the HexEdit window to open them.

 

Searching the Files

 

Once you have the files open you can perform a search using the Find command.  In any of the pages on the Find dialog (Basic Find page, Hex Find page etc), set the Scope to All open files and click the Find Next button repeatedly to step through all occurrences of the search item.  If you just want to mark all occurrences for later then use the Bookmark All instead.

 

Also (if you have background searches enabled -- see System Options page) whenever you perform a new search a background search starts automatically in all open files.  Background searches are performed in low-priority background threads so as not to interfere with the response of the user-interface (the background search thread for the active file has a slightly higher priority but still less than normal).

 

When the background searches have completed you can then flip between the files (see Next Window command) and look at the found occurrences.  The status bar "search occurrences" pane shows the number of occurrences found in the file.  Use the Find Forward command and the Find Backward command to quickly move through all the found occurrences in the active file.

 

For large files the search may take some time.  While a background search is still in progress a progress bar is displayed in the search occurrences pane.
