Refresh Tree Window

 

When you edit the data of a file in the hex or character area of the data window the tree (template) window is not automatically updated to reflect changes.  This is necessary as the tree may take an appreciable amount of time to regenerate, and because incomplete changes may cause a lot of annoying error messages.

 

To refresh the tree view so that it reflects the current data of the file you can use the Refresh Tree button.  Making changes in the tree view will also cause the tree view to be refreshed.  The button is disabled if there have been no changes made to the file since the last refresh.
