Moving Around

 

Every window has a current position marked with the flashing cursor or "caret".  You are probably familiar with how to move the cursor with the cursor movement keys (arrow keys, Pg Up etc).  As with most Windows software, the Home key moves to the first byte on the line, End moves to the last. Ctrl+Home and Ctrl+End move to the start and end of the file. Ctrl+Left and Ctrl+Right move left and right by a whole group of columns.  See Keyboard Shortcuts for more information.

 

The scroll bars allow you to move the current display without changing the cursor position. For a large file the vertical scroll bar (on the right side of the window) is particularly useful. Clicking the arrows at the end of the scroll bar (or using the default Ctrl+U or Ctrl+D keyboard shortcuts) moves by one line. Clicking in the scroll bar above or below the scroll box moves by a page. You can also drag the box to a position anywhere within the file. (Note that the size of the box, in proportion to the size of the entire scroll bar, reflects the amount of the file that is visible within the window – if the entire file is visible then the scroll bar will disappear.) The Ctrl+L (in default keymap) keyboard shortcut scrolls the display so that the cursor is positioned in the middle of the window.

 

If there is no flashing cursor then it has probably been moved offscreen - pressing any cursor movement key will scroll the display so it is visible.  For the same reason if you try to move past the edge of the window the display will scroll to keep the cursor visible.You can force the display to scroll when you get a certain distance from the edge of the display using the "scroll zone" setting (see Document Edit Options).

 

Addresses

 

For each row of the display the left side (address area) shows the address of the first byte of the line.  The address can be in hex and/or decimal (see Document Display Options page).  The ruler and address area highlight the current column and row of the cursor and mouse position -- enabled using the checkboxes in the Ruler section of the Workspace Display Options.  You can also see the location of the cursor as you move it within the file in the Hex Address tool and Decimal Address tool.

 

Finally, you can also see the address of the current byte in a tip window when you hover the mouse over it.  You can configure "Info Tips to show the address in hex and/or decimal (or any radix) or (if you don't specify a format) it uses the current way addresses are shown in the address area.

 

To avoid confusion hex addresses are shown in red, decimal in blue throughout HexEdit.  (The colors can be configured in Color Options page.)

 

Jumping to an Address

 

A simple way to move to a distant address is to drag the scroll box (see Scroll Bars) until the address you want is visible and click within the display to set the cursor location. However, for very large files the scroll bar resolution may mean that you can’t drag to the exact position but may have to use the Page Up or Page Down keys as well.

 

Alternatively, the Decimal Address tool and Hex Address tool in the Edit Bar allow you to jump quickly to a known address within the file. Note that, by default, the "first” byte of the file has an address of zero.  As you type an address in one tool the corresponding address is updated in the other – as a useful side effect, this provides a simple way to convert between hex and decimal numbers. These tools also display the current address as you move the cursor around the screen and during searches and file comparisons.

 

Both of these tools also store separate histories of the last addresses jumped to, which can be selected from the drop down list. Note that you have to press the Enter key to actually jump to the address and return to the edit window. You can use the Tab key to quickly flip between these controls without returning to the edit window.

 

Not only can you type in an address but you can also enter an (integer) expression into the hex and decimal address tools.  The syntax and operators for expressions are almost identical to that used in C/C++.  You can even assign to variables (eg "reclen=10") and use those variables and bookmarks (see below) of the current file to calculate an expression (eg "base+reclen*5").  See Expressions for more information.

 

To quickly jump to any bookmark in the current file select it from the "Bookmarks" drop-down list which appears on the navigation bar.

 

Another powerful (but still fairly easy) way to jump to an address is to use the calculator. If you invoke the calculator using the Edit/GoTo menu item then the current address is loaded into the calculator. You can then type in a new absolute address and click the “GO” button or just press the Enter key to jump to the address. To jump to a relative address click the “+” or “-“ button and type a relative offset and press Enter.  Again you can also enter an expression into the calculator.

 

The Mark, Bookmarks and Highlights

 

A way to return to a previous address is to mark that address with the Mark Position command and later return to it using the Go To Mark command. The mark is also useful for measuring distances between points in the file. There are decimal and hex “Distance to Mark” indicators in the Status Bar that show the distance from the mark to the current cursor position.

 

Bookmarks allow you to associate a name with a file and address.  Use the Bookmarks dialog to add bookmarks, sort them, jump to them etc.  You can also jump straight to a bookmark in any file even if the file is not currently open.  Note that bookmarks store a name plus a file name so bookmarks in different files can have the same name.

 

As you move around a file you can also see the closest bookmark in that file in the Bookmark_tool.  You can also use the Bookmark tool to jump to any bookmark in the file by selecting it from the drop-down list.

 

Another way to remember addresses is to highlight them.  Highlights are sort of like unnamed bookmarks but are much easier to add -- just turn on the highlighter and any selections you make are automatically highlighted.  By default highlights have a background of yellow (bookmarks are blue) but you can change these colors in the Color Options page of the Options dialog.
