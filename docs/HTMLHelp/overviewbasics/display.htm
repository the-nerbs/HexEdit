Documents

 

When you open a file in HexEdit it is, by convention, called a document.  A document can be displayed in more than one window each with more than one view.  However, for now we will just consider the main view (sometimes called the Hex View).  This just displays the raw data of the file in hex, and (optionally) as text.  This is where you view and edit the file.

 

The main area in the middle of the window is the “hex area” which contains pairs of hex digits representing the bytes of the file. To the left is the address of the first byte of the row (displayed in hex or decimal). Optionally to the right of the hex area (or instead of it) is the “character area” which shows the corresponding characters (in the current character set).  There is a fourth mode for displaying the data (apart from hex, character and hex+character) which is called stacked mode where each byte of the file is represented by 3 characters stacked vertically -- at the top is the character (in the current character set) and below that are the two hex nybbles.

 

You can edit the file by changing hex values in the hex area, or by changing characters in the character area.  Editing is discussed in a later section.  For now we will just become familiar with the display.

 

Changing the Display

 

The most common window display options can be adjusted using the Standard Toolbar.  For example, you can select the character set in use from a drop down list, quickly increase/decrease the font size, toggle modes etc.  If you find the colors too garish then select the "Black and White" color scheme from the drop down list. A useful feature is Auto Fit mode where the number of display columns is not fixed but is adjusted to fit the width of the window when it is resized. More options are provided on the Format Bar, such as selecting a new font from the drop down list.  You can also adjust some things, like row size, using the handles on the ruler (discussed later).

 

The Document Display Options page of the Options dialog allows you to change all these display settings plus others like the column grouping.  The Document General Options allows you to save the current settings as the default which is used when a new window is created – ie, when a new file is opened or created.

 

There are four default color schemes (associated with the four character sets that can be displayed), but you can create your own schemes.  To use a new scheme just select it from the Color Scheme Tool on the Standard toolbar.  (You can also associate a scheme with a file type based on file extension).  The scheme determines all colors including background color, highlighter color etc.  It also defines groups of byte values or ranges which are displayed as different colors -- for example, the ASCII default color scheme shows text (32-127) as black, ASCII control characters (0-31) in red, except for special control characters that may appear in text files (8-10, 13) which are green.  The Color Options page of the Options dialog allows you to change the colors of any color scheme and to create new color schemes.

 

Character Sets and Support for EBCDIC and Unicode

 

When text characters are displayed they can be shown in one of four different character sets (ASCII, ASCII+ANSI, ASCII+IBM/OEM, and EBCDIC).  You can select these from the drop-down list on the Standard toolbar.  The character set in use only affects how characters are displayed in the character area not the hex area.

 

The ANSI character set is an extension of ASCII and hence differences are only apparent when viewing characters with the high bit on.  The IBM/OEM character sets is the set of characters that were used for the video display of the original IBM PC and compatibles -- when displaying using the IBM/OEM character set a different set of fonts (than the Windows ANSI fonts) is available to be used.

 

The EBCDIC character set is supported by translating to an ANSI font (using the file called EBCDIC.TAB).  In EBCDIC mode searches are performed using the EBCDIC character set, and copying/pasting to the clipboard translates to/from ASCII as appropriate.

 

Note that the display of the Unicode character set is not currently supported in the character area.  This is mainly due to the display and editing being byte oriented (especially in stacked mode) whereas Unicode consists of double-byte characters.  (If there is demand then we will consider allowing Unicode display/editing in the character area.)  However, you can always see the current Unicode character in the Byte Properties Page.  Unicode is also supported in other areas such as searches and pasting from the clipboard.

 

Windows and Files

 

HexEdit allows you to edit multiple files simultaneously, and you can open more than one window on the same file. For example, you might want to make changes to the end of a file, while viewing the start of it. To do this, open a window for the same file and scroll one to the end of the file and start making the changes. You can even make the other window read only (see Allow Changes Toggle) so that you don’t accidentally make changes in it.

 

Every window in HexEdit has a set of attributes associated with it that can be set independently. When you reopen a file the same attributes are restored, so the window looks exactly like it did when you last worked on it.  These attributes are kept in the "recently used file list" so if you remove the file from the list of recently used files then this information is lost (see Recently Used Files Dialog).

 

Information remembered includes the cursor position and how the file is displayed -- the number of display columns (default 16) and the column grouping (default 4), which areas (hex and character) are displayed, what character set to use in the character area (ASCII, EBCDIC, etc), the font, color scheme and other display options.

 

You can also configure a window to use overtype or insert mode, and toggle between read only and modifiable modes.

 

Each window also has an associated “mark”, which allows you to mark a position, return to it, measure distances from it, etc. You can also highlight a group or several groups of bytes using the Highlighter.

 

Global Options

 

Most display options are controlled separately for each window – you can use a different font etc in each window. There are, however, several global options that change the display and editing behavior of all windows. For example, whether the ruler is displayed is a global option (see Workspace Display Options); how undo works is set for all files in Workspace Editing Options.
