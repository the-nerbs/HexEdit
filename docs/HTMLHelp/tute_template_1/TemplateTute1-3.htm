Step 3. Saving the Template

 

         

Now would be a good time to save the template.

 

1. Right-click the root tree node (the top node containing the word "Default") and select the "Save As..." menu item.

 


 

Figure 3.1 Select the "Save As..." menu item.

 

You can use any file name.  Template files have the extension ".XML" but that will be added automatically if you don't type it.

 

If you want a template to always be associated with a certain type of file give it a name starting with an underscore followed by the file extension used for that type of file.  For example, if you want the new template associated with all .TXT files then give it the name "_TXT.XML".

 

2. Give the file a name of "_TXT" and click the "Save" button.

 



Figure 3.2 Save the template with the name "_TXT".

 

Modifying the Template

 

There are still a few minor problems with the template which we will now fix.

 

The template name (displayed in the root node) is different to the file name.  The template name is still "Default" so we should change it to something more meaningful.

 

3. Right-click the root node of the tree and select the "Edit..." menu item, which brings up the "STRUCT" dialog.

 



Figure 3.3 Edit the root node.

 

You can also just double-click a node to edit it.

 

4. Change the name from "Default" to something like "ASCII_Text_File" in the STRUCT dialog.

 

A "STRUCT" is just a sequence of other elements (FOR, DATA, STRUCT etc).  You can add STRUCTs throughout a template.  The root node is just a special case of a STRUCT which contains all other elements of the template.

 

Under the "Name" filed is the main part of the STRUCT dialog - a list-box showing all the elements contained in the STRUCT.  To the right of the list are buttons to change the entries.  You can edit, insert, delete and reorder the entries in the list.

 

To append an element to the end of the list select the empty entry past the last element in the list and click the Insert button.

 

Like the FOR dialog the STRUCT dialog contains "comments" and "type name" fields which you can put meaningful information into.  The "display" field is used to display information next to the template name in the root node - see STRUCT Dialog for more information.

 

The list of elements should currently have just 2 entries: one "FOR" element for the lines of text; and one DATA element called "none".  We no longer need the "none" element.

 

5. Select "none" in the list and click the Delete button (which contains a red X) and click OK to confirm the deletion.  Then click OK to close the STRUCT dialog.

 

 



Figure 3.4 STRUCT dialog.

 

Saving the Template

 

You can easily save you changes at any time by right-clicking the root node and selecting the "Save" menu item.  When you close the data file the associated template is also closed.  If you try to close without saving the template HexEdit will prompt you to save it.

 

6. Close the text file, and when HexEdit prompts you click the Save button.
