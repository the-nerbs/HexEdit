Step 2. Creating the Template

 

         

We will now create a new template by modifying the default template.

 

Adding a DATA Element

 

First we will add a DATA element that represents a line of text from a text file.

 

1. Right-click the "none" element to display the popup menu and select the "Insert" item and then the "Data..." item from the sub-menu.

 

This will insert a new DATA element before "none" and open the "Data Element" dialog.

 



Figure 2.1 Adding a DATA element.

 

2. In the dialog give the DATA element a name of "line".  Select a "type" of string and a "format" of ASCII (or Unicode if you opened a Unicode text file).

 

The name can later be used in expressions (although in this case it won't), so it must be a valid identifier.  In particular, it should not contain spaces.

 

Next we have to indicate how long the string is.  If a "length" is given then that many bytes is always used (although the value of the string may be shorter if a terminator is encountered).  If a length is not given then the length of the string is all the bytes up to and including the terminator character.  If no terminator character is specified then it defaults to a zero (null) byte which is often used to terminate strings (in particular in C/C++).

 

However, for our text file template the string is a line of text which can be any length and is terminated by a CR-LF pair (carriage-return plus line-feed) or just a line-feed for UNIX text files.

 

3. Leave the "length" field blank and enter 10 (line-feed) into the "terminator" field.

 



Figure 2.2  Each line is terminated by a line-feed.

 

The "Comments", "Type name" and "Units" (for numeric fields) have no effect on the template processing.  You can put some explanation in the "comments" field now if you like.

 

When a binary file is created or modified by a C/C++ program the "Type name" field is useful for storing a type name (such as a typedef, struct or class name).

 

The "Domain" field can contain a boolean expression used to check whether the field is valid.  We will see an example of this a little later.

 

The "Read-only" checkbox can be used to allow or disallow modifications to specific data fields of the file.  By default it is in a 3rd (unspecified) state which indicates that the read-only status is determined by a global read-only status for the whole file.

 

Making the whole file read-only then turning off the read-only status of a few fields allows you to easily control which bytes the user of the template can change.

 

The "Big-endian" checkbox is only enabled for numeric fields.

 

4. Click OK to add the new DATA element.

 

Adding a FOR Element

 

We now have a DATA element that represents a line of text, but most text files have more than one line.  We will add a repetition (or array) of the line by adding a "FOR" element and making the "line" DATA element its sub-element.

 

5. Right-click the "line" element to display the popup menu and select the "Enclose in" item and then the "For..." item from the sub-menu.

 



Figure 2.3 Enclosing the "line" DATA element in a FOR element.

 

We will give the "FOR" Element the name "line".  This will allow you to access an element of the "array" in expressions using an index.  The first element is "line[0]" and so on.

 

Use the singular when giving an array a name.  HexEdit automatically add an "s" to make the plural when displaying in the tree-view.  It also make expressions more readable when you use something like "line[n]".

 

Sub-elements of a "FOR" element do not require a name since they are accessed via an index into the array.  You can delete the name of the "DATA" element now to avoid confusion by clicking the "Edit" button.
 

The length of the array is determined by the "count" and/or "stop" fields.  However, in this case we are going to leave them empty to indicate that the array goes all the way to the end of file.

 

6. Enter "line" for the name and click OK.

 



Figure 2.4 FOR dialog.

 

Note that the tree has been updated to include the FOR.  Expanding the FOR node displays several lines, one for each line of text in the file you have open.

 

Note also that now the "none" element icon has changed to show that the element has zero size.  This is because the array of lines goes all the way to the end of file.  We will delete it in the next step of the tutorial.
