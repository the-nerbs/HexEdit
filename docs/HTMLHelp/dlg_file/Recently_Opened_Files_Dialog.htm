Recently Opened Files Dialog

 

 

HexEdit stores the name and other information about all files that you have opened.  You can also see the top few entries in this list in Recently Opened Files entries of the File menu.

 

You can resize the dialog if you want to see more or less of the list.

 

List of Recently Opened Files

Most of the dialog is taken up with the list which shows all of your recently opened files.  The list is split into (up to) 9 columns (described below).  You can show and hide any of the columns by right-clicking any column heading and selecting from the list.  You can also resize any of the columns by dragging the separator (the vertical line between the column headings), or double-click the separator to size the column to the width of the widest item.

 

By default the files appear in the order they were opened, with the most recent at the top, but you can sort on any column by clicking on the column header.  Click the same column header again to reverse the order.

 

To select a file to open or remove from the list just click anywhere on its row.  You can select more than one entry by shift-clicking to select a contiguous group or by control-clicking to select or deselect individual entries.  The number of files currently selected is shown in the Selected count underneath the Remove button.

 

The columns of the list are:

 

Column

Description

File Name

The name of the file (excluding path information which is shown in the Location column).

Location

This is the folder (directory) in which the file is located.

Last Opened

This is when you last opened the file in HexEdit .  This is not to be confused with the time of last access (see below) stored by the file system.

Size

This is the size of the file on disk.

Type

This is a brief description of the file (based on the file extension) according to Windows.

Modified

This is when the file was last modified according to the filesystem.  This will be similar to the last opened time if you last changed the file in HexEdit.

Attributes

Shows the characters: R (read-only), A (archived), H (hidden), S (system) which are flags stored by the file system (see File Properties Page for more info)

Created

This is when the file was created according to Windows.  For FAT filesystems this may not be stored and may be the same as Modified time (above).

Accessed

This is when the file was last accessed according to Windows.  For FAT filesystems this may not be stored and may be the same as Modified time (above).

Category

A category can be associated with every file (see File Summary Page).  It is useful to categorize files so you can sort by this column.

Keywords

One or more keywords associated with the file.

Comments

Any other free-form text to be associated with the file.

.

Open

Clicking this button opens all selected files and closes the dialog.  If there is no file selected in the list then this button is disabled.  You can also double-click an entry in the list to open just that file.

Open read-only

When the file(s) are opened they are opened in read-only mode.

Open shareable

When the file(s) are opened they are opened in shareable mode.

Close/Save

This closes the dialog.  If items have been deleted (using the Validate or Remove buttons) then this button changes to Save and selecting the button removes the items from the list.

X

Select the Close button in the top-right corner to close the dialog without making any changes.  Any items removed from the list using the Validate or Remove button are not removed from the list.

Remove

This marks selected file(s) for removal from the recently opened files list. The files are not actually removed until you hit Enter or click the Save button; if you change your mind just close the dialog or press Escape.

Removing a file loses formatting (font, the number of columns/offset/grouping, autofit, decimal addresses etc), display (window position, hex/char area display), editing (RO/RW and OVR/INS modes) and other information (highlights, mark etc) associated with the file.  (This information is stored with the entry for each recently opened file.)  However, bookmarks for the file(s) are retained.

The disk file(s) themselves are not affected by removal from the list.

Validate All

Use this to check the validity of the files in the recently use list.  If a file is not found on disk then it is removed from the list. The files are not actually removed until you click the Save button; if you change your mind just close the dialog or press Escape.

Keep network and removeable files

When validating you may not want to remove files from the list if they are not currently available.  For example, a file may reside on removeable media (floppies, zip disks, CDs, DVDs etc) or on a network drive that is not always available.  When this check-box is on and the Validate All option is used only files on fixed local drives will ever be removed from the list (if not found).

Search

Allows searching all fields of the list (case-insensitive).  All items found are selected.

Help

Click the Help button to display this information in the on-line help.
