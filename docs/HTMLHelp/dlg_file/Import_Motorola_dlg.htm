Import Motorola S Records dialog

 

 

This dialog is invoked with the Import Motorola S Records command.  It is based on the standard File Open dialog.  Double-clicking a file (or selecting and clicking the Import button) imports the file.

 

Look In

To see how the current folder fits in the hierarchy on your computer, click the down arrow. To see what's inside a folder, click it.

File Name

Type the filename you want to open. If you type the name of a folder that folder is opened and the contents are displayed. Normally you see all files matching the specification currently selected in the “List Files of Type” box – but if you type a filename with wild-cards here, such as *.S1, you will only see matching files.

List Files of Type

Select the type of file(s) you want to open.  By default Motorola S files (*.s) are displayed, but you can elect to use a different filter by selecting from the drop down list.  You can change the default filter (*.*) and change, add or delete from the filter list by using the Filters page of the Options dialog.

Use Import Addresses (for discontiguous records)

If this option is off then HexEdit will import from the file at the current position in the file.  But in this case the Motorola S records in the file must be contiguous otherwise the import stops with an error.  Also the imported bytes are always inserted into the file (you must be in INS mode) at the current cursor location.

 

If this option is on then the records do not need to be contiguous (in fact, there are no restrictions at all on the expected addresses).  Each record's data is added to the file at the specified record address.  In other words the existing data of the file is overwritten (even in INS mode).  If an address is past the end of the file then it is ignored.  The data replaces the data of the file, hence the file length never changes as a result of the operation.

Highlight

If discontiguous records are imported it is often difficult to see which bytes were changed.  This option highlights all imported bytes (if the above option is used).  Note that all existing highlights are lost.
