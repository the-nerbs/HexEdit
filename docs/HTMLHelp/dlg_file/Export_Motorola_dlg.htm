Export Motorola S Records dialog

 

This dialog is invoked with the Export Motorola S1 Records, Export Motorola S2 Records, and Export Motorola S3 Records commands.  It is based on the standard File Save dialog. You can enter a name and select the location of the file you want to export to.

 

The folders and files in the current location are displayed. Double-clicking a folder opens it – that is, your current location moves to that folder. To open the folder one level higher (the “parent” of the current folder) click the “Up One Level” button. Double-clicking a file will attempt to replace that file – you are prompted if you want to replace it.

 

Save In

To see how the current folder fits in the hierarchy on your computer, click the down arrow. To see what's inside a folder, click it.

File Name

Type the filename you want to export to. If you type the name of an existing file you are prompted if you want to replace it. If you type the name of a folder that folder is opened and the contents are displayed.

Save As Type

By default you only see existing files with a .S file extension; this is commonly used as the file extension of Motorola-S format files.  To see other files select a different filter from the list.  To see all files use the *.* filter.  Note that you can add, delete or modify the filters list by using the Filters page of the Options dialog.

Export (discontiguous) highlighted areas only

If this option is selected then only the currently highlighted bytes in the active file are exported (see Highlighter).  Otherwise the currently selected bytes are exported.

 

When this option is on each S record is written with its actual file address.  For example, if you import a Motorola S file using the "Use Import Addresses" and the "Highlight" options (see Import Motorola S Records dialog) then subsequently export the file using the "Export highlighted areas only" option then you will get back the same logical file (unless some of the original records were past the end of the file imported into).

 

This option is disabled if there are currently no bytes highlighted in the active file.
