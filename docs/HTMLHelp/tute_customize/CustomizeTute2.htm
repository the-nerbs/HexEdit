Step 2. Keyboard and Mouse Customization

 

         

 

Customizing the keyboard can be a great time-saver especially in situations where you are not using the mouse.  On the other hand when you are WMAH (with mouse at hand) it is very convenient to assign an oft-used command to the mouse double-click

 

In this step of the tutorial you will learn how to customize the keyboard and mouse double-click events.

 

 

Assigning a Keyboard Shortcut

 

AutoFit is a very useful command that by default is not assigned a keyboard shortcut.  We will assign it to Ctrl+A.  But since Ctrl+A is already assigned to the "Select All" command we have to unassign it first.

 

1. Open the Customize dialog and select the Keyboard (4th) tab.

2. In the "Category" drop-list select "Edit" to show the commands on the Edit menu.

3. In the "Commands" list below that click "Select All" near the end.  Ctrl+A should now show in the Current Keys list.

4. Select "Ctrl+A" which is the only entry in the Current Keys list and click the "Remove" button.

 

 

Ctrl+A is no longer assigned to the Select All command.

 

5. In the "Category" list select "View" to show the commands on the View menu.

6. In the Commands list select "Auto Fit", near the top.

7. Move to the "Press New Shortcut Key" edit box by clicking on it and type Ctrl+A.

 

The edit box should now say "Ctrl+A", as in the screenshot below, and beneath that it should say "Assigned to: [Unassigned]".  If the keystroke is not unassigned check your steps above or find the command that it is assigned to and remove the assignment.

 

 

8. Click the Assign button and click Close to close the Customize dialog.

 

Note that any keyboard mappings set up this way are instantly reflected in menu text and toolbar button tips.  If you check on the View menu to the right of the item for "Auto Fit" you should see the text "Ctrl+A".  Also if you hold the mouse over the AutoFit button on the Standard toolbar the popup tip that shows should say "Auto Fit (Ctrl+A)" (unless you have turned off "Show shortcut keys in screen tips" in the Customize Dialog Options page).

 

 

Printing a List of Key Assignments

 

All current keyboard assignments can also be seen in the keyboard map.  This is useful if you have highly customized the keyboard, or you just want to see the default keys for all the commands HexEdit provides.  From the Keyboard Map dialog you can also print the list or copy it to the clipboard.

 

9. Open the map by selecting Keyboard Map from the Help menu.

10. From the "Category" list select "All commands".

 

You can sort the list in different ways by clicking on a column heading.  You can resize the columns by dragging the column dividers in the headings.

 

 

11. Click the printer button at the top-left to print the list.

 

As the above was just an example you might want to set the keyboard mappings back the way they were.  The Ctrl+A keyboard shortcut is a de facto standard in Windows and it might be better to assign it back to the Select All command.

 

You can set all keyboard mappings back to the "factory default" using the "Reset All" button in the Keyboard tab of the Customize dialog.  Of course, if you have other keyboard shortcuts assigned they will be lost.  The alternative is to just reassign the Ctrl+A key manually.

 

 

Assigning Mouse Double-Clicks

 

Currently (in HexEdit 3.2) there are five double-click events that can be assigned.  By default these are assigned to:

 

Area

Double-click Effect

Address Area

selects all bytes on the same line as the address

Hex Area

moves the mark to the position clicked

Character Area

moves the mark to the position clicked

Bookmark

opens the Bookmarks dialog

Highlight

selects the highlight

 

Let's re-assign the address area double-click event to toggle between displaying hex and decimal addresses.

 

12. Open the Customize dialog and select the Mouse tab.

13. In the "Views" list on the left select the top entry (Address Area)

14. On the right make sure "Associate double-click event with" is selected.

15. Select View/DecimalAddresses in the list.  The description at the bottom should say "Use hexadecimal or decimal addresses".

 

 

16. Close the dialog and test the result by opening a file and double-clicking the address area.
