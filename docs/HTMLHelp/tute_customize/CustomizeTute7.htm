Step 7. Miscellaneous Customizations

 

         

 

In the final step of this tutorial we will look at some other simple customizations.

 

 

Automatic Macros

 

By giving macros a special name they can be run automatically.  If you save a macro using a name for a specific file extension then that macro will be run whenever a file with the extension is opened.  For example, if you save a macro to a file called "_EXE.HEM" then whenever an .EXE file is opened in HexEdit the macro is run.

 

This facility can have many uses -- as an example,  for a certain file type you might always want to position the insertion point after the header at the start of file.

 

Another special macro called "AUTOEXEC.HEM" is run whenever you start up HexEdit.

 

To demonstrate we will create a macro for .HEG files that jumps past the header at the start of the file.  The (fictitious) .HEG file format has a 16 byte header.

 

1. Open any file, to allow recording of macro commands that operate on a file.

 

2. Select Edit/Go To which brings up the Calculator, and click the "Hex" radio button if not already in Hex mode.

 

We open the calculator before macro recording is started so that it is not recorded as part of the macro.  We need the calculator open as it is used to jump to address 16 (10 hex) in the file.

 

3. Select Tools/Record Macro (or press F7, or double-click the "REC" pane on the status bar) to start macro recording.

 

 

3. Enter "10" in the calculator and click to "GO" button.

 

4. Select Tools/Record Macro (or press F7, or double-click the "REC" pane on the status bar) to stop macro recording.

 

5. To test that the macro works move the cursor elsewhere and press F8 to run the macro.

 

Now we just need to save the macro we have created to disk.  We give it the file name "_HEG.HEM" so that it is run when any .HEG file is opened.  Note that the first character of the file name (before HEG) is an underscore - this denotes that the macro file is special.

 

6. Open the Options dialog (Tools/Options) and select the "Macro" tab.

 

7. Click the "Save Current Macro" button and name it "_HEG".  Add a comment to the macro if you like.

 

HexEdit adds the ".HEM" file extension automatically for HexEdit macro files so you do not need to type that.

 

 

8. Click the Save button to save the macro then click OK to close the Options dialog.

 

9. Open the file called "TUTORIAL.HEG" which you can find in the folder where HexEdit was installed. Verify that the cursor (insertion point) jumps to the 16th byte.

 

This macro jumped to a fixed address but you can record virtually any sort or calculation in macros, including reading and manipulating values from the active file.  See Keystroke Macro Tutorial for an example.

 

 

Default Display

 

HexEdit remembers the display options for all files you have ever opened.  When you open a file that you have previously opened, all the formatting options are restored exactly as they were when you last used it.

 

But when you open a file that you have never opened before the default display settings are used.  These are also used when you create a new file in HexEdit.  You can easily change the default settings as shown here.

 

10. Open any file.  We need a file open to modify (and preview) display settings.

 

11. Open the Options dialog (Tools/Options) and go to the Document General page.

 

The Document pages of the Options dialog are not shown if you skipped the previous step and do not currently have a file open.

 

 

 

12. To see the current default display settings click the "Use Default Settings" button, in case some of the settings for the open file have already been altered.

 

13. Change to the Document Display page and turn on the Maximize checkbox.

 

 

 

See the Window Display Options page and Document Edit Options for details on the different document settings.

 

14. Click the Font button to change the default font used.

 

The factory default font is "Courier", which is not a scalable font.  This font is chosen as it is a fast, nice-looking monospaced (non-proportional) font available on all versions of Windows.  If you prefer a scalable font then Lucida Console is a nice monospaced font available on newer versions of Windows.  A scalable font gives better results when you change the font size (eg by using the mouse wheel while holding down the Ctrl key).

 

 

To preview the display settings changes at any time just click the "Apply" button to see how they look in the active file.

 

15. Go back to the Document General page and click the "Save Current Window Settings as Default" button.  Click the Cancel button to close the Options dialog.

 

16. Create a new file or open a file that you have never opened before in HexEdit and verify that your new default settings are used.

 

 

Printing

 

A very nice feature of HexEdit is the ability to customize how pages are printed.  In this section we will see how to change the header, footer and margins.

 

17. Open the Options dialog (Tools/Options) and go to the Printing page.

 

18. Select the units you prefer (Inches or Cm) and adjust the margins of the page.

 

The left and right margins are just minimums and may actually print larger than requested.  For example, if the line length on the screen is only half the page width then the margins will take up half the page width.  On the other hand for long lines the line may be wider than the page in the current font size - in this case HexEdit automatically reduces the font size until the text fits within the margins of the page.

 

19. Adjust the distance that the header is from the top of the page and the distance the footer is from the bottom of the page using the "From edge" settings in the bottom right corner.

 

The header is printed within the top margin and the footer within the bottom margin.  If there is not enough room within the margin then it is simply not printed.  For example, the top margin must be greater than the Header's "From edge" distance plus the height of the text.

 

20. Turn on the options (checkboxes) to draw a border and column headings.

 

Now we will set up what is printed in the header and footer.

 

First note that both the header and footer are divided into three parts: left part (aligned with the left side of the page), center part (centered horizontally on the page) and the right part (aligned with the right side).  To specify all 3 parts you need to separate them with a special character -- the vertical bar (|).

 

As well as normal text you can also use substitutable parameters which are specified using an ampersand (&) followed by a letter.  See the Printer General Options page for a complete list of substitutable parameters.

 

You can put any text into the headers and footers except for vertical bars which are used to separate the parts.  To show an ampersand (&) use two of them (&&).

 

21. In the Header field enter "&A | CONFIDENTIAL | &M". This prints the full file name in the top left of the page and the file modification date/time in the top right, with the word CONFIDENTIAL in the center.

 

22. In the Footer field enter "Printed: &N | | Page: &P".  This prints the date/time the file will be printed in the bottom left and the page number in the bottom right, with nothing in the center.

 

 

 

23. To see how the page will look open a file then invoke print preview (File/Print Preview).

 

If you have a long header or footer (such as the example header above) and use a large font the text in the 3 parts may overlap.  Luckily headers and footers are printed using the same font as the rest of the page.  So you can avoid this problem by reducing the size of the font used to display the file (see Window Display Options page).

 

24. Close print preview, reduce the font size (eg click the Decrease Font button on the Standard toolbar), then check the effect using print preview again.

 

 

Conclusion

 

That brings us to the end of the customization tutorial  I hope you have found many useful tips and will continue to look for ways to increase your productivity and enjoyment of the software.

 

There are many other ways you can customize HexEdit.  You might find some more ideas in the Github Discussions of our on-line forums.  One of the more obscure facilities is the ability to configure how ECBDIC characters are displayed, searched for etc -- this is all handled by a translation file called "EBCDIC.TAB" which is stored in the same place HexEdit is installed.  You can simply modify this file (in HexEdit, of source) to display EBCDIC (or any type of characters) the way you want.
