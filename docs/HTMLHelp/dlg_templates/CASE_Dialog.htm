CASE Dialog

 

 

This dialog allows you to set the options of a multi-way branching element (often called a switch or case statement in programming languages).  This edits a SWITCH element containing one or more CASE elements in the underlying XML code.

 

The element is similar to an IF element in that it conditionally chooses among alternatives.  However, instead of selecting on a Boolean (true or false) condition it selects based on an integer expression.  The result of the expression searches through the list of cases.  Each case has a value, or range(s) of values, that are checked against the expression plus a sub-element, of any type that is used if the expression matches.

 

The list of cases is searched in order until a match is found.  Only the first one found is used even if more than one matches so the order is important. There is no default case (like the C/C++ default case in a switch) but can easily be emulated by creating a case that matches all possible values, by using a range like "-4000000000 - 4000000000" - of course, this CASE should be last in the list.

 

Like IF elements there is no name.  For expression evaluation, the name of a sub-element of a CASE can be used in expressions.  The only valid name is the taken CASE element (ie the one that matched the expression) but typically all CASE sub-elements would use the same name.

 

Expression

The expression can return an integer or string result (new in 3.2).  The expression can use any value from the template so far processed.  Use the button with the triangle to see a menu (and possibly submenus) of all available values.

 

The expression is used to scan the cases from top to bottom until a match is found.  For a string expression an exact (case-sensitive) match is required; for an integer expression the result can be matched against a single value or a list of values or ranges as explained below.

 

Only the matching CASE sub-element is shown in the template tree view unless you are in edit mode. In edit mode the other CASE elements are shown (grayed out) allowing you to edit them.

 

Cases

List of CASE elements.  Select an element in the list before changing the range or using the buttons next to the list (Edit etc - see below).  Select the blank entry at the bottom of the list before inserting in order to append to the list.

 

Integer/range(s) or Match String

Each case has a string against which it is matched.  Select the case in the list to change how it is matched.

 

If the expression returns a string result then a case-sensitive string comparison is perfomed.  Please make sure that the match string does not contain trailing spaces to avoid confusion.

 

If the expression returns an integer result then the match string can be a single integer or a comma-separated list of integers or ranges.  For example, "1, 4, 9, 100-200" will match 1, 4, 9 or any number between 100 and 200 inclusive.

 

The last case can have an empty match string in which case it matches everything (ie anything not matched by a case earlier in the list).  This is like the default case in a C switch statement.

 

Edit...

This edits the CASE sub-element that is currently selected in the Cases list.  A "child" dialog appears near the dialog - you can't do anything further in the CASE dialog until the child dialog is closed again.

 

Insert

Inserts a new CASE element in the list just before the currently selected element in the Cases list.  To add an element to the end, select the blank line in the list past the last one before clicking the Insert button.

<< and >> buttons

If the parent element is a STRUCT then these buttons are displayed in the bottom right of the dialog.  They allow you to move to the previous and next sibling (ie the previous and next elements in the parent STRUCT).  This is very handy when you quickly want to view or change values in all the children.

 

Using the button is equivalent to clicking OK then editing the previous or next element in the parent STRUCT.
