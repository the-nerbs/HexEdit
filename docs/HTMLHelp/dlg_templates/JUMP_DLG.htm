JUMP Dialog

 

 

This dialog allows you to set the options of a template JUMP element (and its contained element, using the Edit button).  It is usually invoked by right-clicking the JUMP node in the template tree view and selecting the "Edit..." item.

 

A JUMP element allows "out of order" processing of data in a file.  This is typically used to handle a "header" at the end of file. It is also useful as a "look ahead" facility as after processing the JUMP element the current file position returns to the position before the JUMP.

 

A JUMP element does not have a name but takes the name of its contained element, for the purpose of expression evaluation.

 

Address

An expression used (along with the Relative to field) to determine the new data file address.

 

Relative to

The new address can be relative to start or end of file or the current position.

 

Element

This displays the name and/or type of sub-element contained in the JUMP (typically a STRUCT).

 

Edit

Use this button to edit the element of the JUMP.

 

Replace with

Use this button to replace the element contained in the JUMP with an element of a different type.

<< and >> buttons

If the parent element is a STRUCT then these buttons are displayed in the bottom right of the dialog.  They allow you to move to the previous and next sibling (ie the previous and next elements in the parent STRUCT).  This is very handy when you quickly want to view or change values in all the children.

 

Using the button is equivalent to clicking OK then editing the previous or next element in the parent STRUCT.
